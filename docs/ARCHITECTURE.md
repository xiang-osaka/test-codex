# 情报信息会员 App - 技术架构概要

## 1. 目标
- 支撑多数据源采集、结构化入库、可搜索与订阅。
- 支撑会员权益、计费与权限控制。
- 可扩展到企业版与 API 服务。

## 2. 系统组件
### 2.1 数据采集与处理
- **采集器**：RSS、公开 API、网页抓取、第三方数据供应商。
- **清洗与解析**：去重、实体识别（公司/行业/人物）。
- **标注与摘要**：标签系统 + 摘要模板。

### 2.2 后端服务
- **内容服务**：情报库、检索、订阅。
- **会员服务**：等级权益、支付状态、额度校验。
- **通知服务**：推送、邮件、Webhook。

### 2.3 数据存储
- **主库**：PostgreSQL（用户、订单、内容元数据）。
- **搜索**：Elasticsearch / OpenSearch（全文检索）。
- **缓存**：Redis（热数据、订阅任务队列）。
- **对象存储**：报告文件与附件。

### 2.4 前端与客户端
- **App**：iOS/Android（React Native / Flutter）。
- **Web**：管理端与报告展示。

## 3. 关键数据模型（简化）
- **User**：id、邮箱/手机号、会员等级、到期时间。
- **IntelItem**：id、标题、摘要、来源、标签、发布时间。
- **SubscriptionRule**：id、用户、关键词、行业、通知频率。
- **Report**：id、标题、类型、文件地址、权限等级。
- **Order**：id、用户、金额、支付状态、周期。

## 4. 权限控制策略
- 请求拦截层校验会员等级与剩余额度。
- 高级内容与下载接口进行签名验证。

## 5. 监控与分析
- 采集失败率、延迟监控。
- 用户活跃与转化漏斗。
- 关键事件埋点：收藏、订阅、下载。
